## 三、子系統拆解與建議排序

- **B1 雙 Agent 閉環** —— **第一優先**。風險最低、離 demo 最近：diagnose 半套已在，只要「回寫 prompt」就能秀出「玩偶自己調難度/換主題」＝自動迭代最小可行展示。
- **B2 長期記憶 profile** —— 第二。是 B1（directive 需知「已會什麼、興趣」）與 B3（難度落點種子）的共同資料底座。
- **B3 CEFR 難度階梯** —— 第三。需要 B2 的字彙掌握/興趣資料才紮實；把 B1 的 difficulty 從「趨勢粗判」升級成「可計算的 25 微階 + 升降級規則」。
- **B4 隱私 / Guardrails** —— 全程橫切，隨 B1 引入雲端 provider 時同步補（system prompt 護欄、雲端輸出過濾、去識別化）。

**共同介面**：三者都收斂到 `diagnose.generate_diagnosis` 的回傳 dict 新增欄位（`companion_directive` / `level_state`），經 `store.add_diagnosis` 的 JSON payload 天然相容落地（**零 migration**），由 pipeline 快取後在即時路徑注入。彼此可獨立實作、獨立測試。

## 四、B1 雙 Agent 閉環：spec 級實作規劃（最詳）

### 4.1 目標展示
連續幾輪流暢 → 玩偶自動升句型；卡住/沉默 → 玩偶自動降成二選一/單字。**不引入 agent 框架**，沿用 diagnose 現有雙軌。

### 4.2 directive 資料結構（新增，掛在診斷 dict，與 `instructions` 平行）
```python
companion_directive = {
    "level": "L2",                       # 四維平均映射（展示用，B3 就緒後改吃 level_state）
    "difficulty": "up" | "hold" | "down",
    "next_goal": "引導用顏色形容詞描述動物",
    "topic": "農場動物與顏色",
    "example_questions": ["What color is the pig?", "Do you see a cat?"],  # 1–3 句
    "fallback_hint": "若學生沉默或困惑，退回二選一：Is it red or blue?",
}
```
- **規則式產生**（demo 主線、零依賴）：重用 `diagnose._detect_patterns` 的 `flags`（`diagnose.py:144`）、`scores`、`prev`。difficulty 重用 `_build_emotional_status` 的趨勢 delta（`diagnose.py:214`：delta≥+1.5 且 avg≥60→up；≤−1.5→down；否則 hold）。goal/topic/questions 由最高優先 flag 決定（與 `_build_instructions` 分流一致，`diagnose.py:233`）。
- **有 API key 走 Sonnet**：在 `_call_anthropic_api`（`diagnose.py:355`）的 `schema_example`/prompt 加 directive 範例；`_validate_diagnosis`（`diagnose.py:315`）**軟性正規化**（缺失/格式不符不丟例外，由呼叫端補規則式）。

### 4.3 最小改動清單（4 檔）
| 檔案 | 位置 | 改動 |
|---|---|---|
| `server/diagnose.py` | 新增 `_build_companion_directive` / `_level_for` / `_avg` / `_normalize_directive` / `format_directive_for_prompt` | 規則式產 directive + dict→prompt 文字化 |
| `server/diagnose.py` | `_rule_based_diagnosis`（:296）、`_validate_diagnosis`（:315）、`generate_diagnosis`（:427） | 回傳加 `companion_directive`；軟性正規化；**收斂保底**（不論 API/規則式，最終一定有合法 directive） |
| `server/llm.py` | `EdgeLLM.generate`（:102，prompt 在 :114） | 加 `directive: str \| None = None` 參數，注入 user_prompt；**護欄不變**：target 仍由 scaffold 決定，`llm.py:144-146` 保底補帶讀句 |
| `server/pipeline.py` | `__init__`（:111）、`_process_text`（:179，呼叫在 :205） | 加 `_directive`/`_turn_count` 快取；傳入 generate；`_refresh_directive` 背景 task（每 N=5 輪，`asyncio.create_task` 不 await） |
| `server/app.py` | `/api/network_mode` cloud 分支（:146 後） | 單行：把新診斷 directive 推進 `pipeline._directive` |

### 4.4 異步更新時機（導師絕不進即時路徑）
1. **每 N=5 輪**：`_process_text` 回覆送出後 `create_task(self._refresh_directive())`（讀 DB→產診斷→存 DB→更新記憶體快取，全在 `asyncio.to_thread`）。
2. **切 cloud 時**：`app.py` 本就產新診斷，直接把 directive 塞進快取。
3. **session 結束**（可選）。
- 即時路徑**只讀記憶體字串** `self._directive`，零 DB、零網路、零阻塞。

### 4.5 韌性總表（directive=None → 全部退回現況）
| 情境 | 行為 |
|---|---|
| directive=None（冷啟動） | prompt ＝ 現況，行為不變 |
| 導師背景更新失敗 | 保留舊 directive，即時路徑照跑 |
| Sonnet API 失敗 | fallback 規則式，directive 由保底補上 |
| directive 帶偏 LLM | target 仍由 scaffold 決定，保底補帶讀句 |
| LLM 整體不可用 | 既有降級用 `scaffold.reply_text`，directive 無害 |

### 4.6 demo 驗收腳本（如何秀出「玩偶自動調難度」）
1. 用 `store.seed_demo()`（`store.py:209`）灌入阿明 14 天曲線資料。
2. 連續幾輪餵「完整、流暢、主動擴句」的 student_text（可用 `run_turn_text` 走文字輪跳過 ASR）→ 過 N=5 輪觸發 `_refresh_directive` → 觀察 directive `difficulty` 變 `up`、下一輪玩偶回覆帶入更複雜句型/新話題。
3. 改餵「沉默/I don't know/單字中文」→ directive 轉 `down`、玩偶退回二選一。
4. 斷網（無 API key）重跑 → 規則式 directive 仍生效，證明離線可用。
5. 驗證點：diagnosis payload 含 `companion_directive`；`EdgeLLM.generate` 的 user_prompt 含「本輪教學策略」區塊（stub model 攔截 messages 斷言）。

### 4.7 新增測試建議
- `diagnose`：輸出含 `companion_directive`；趨勢決定 difficulty（up/down/hold）；API 缺 directive 時保底補上。
- `llm`：`generate(..., directive=None)` 與現況一致；帶 directive 時 prompt 含策略區塊。
- `pipeline`：`_directive=None` 走現況；設定後傳入 generate；`_refresh_directive` 不阻塞即時路徑。

## 五、B2 長期記憶 profile（詳見 `b_axis/B2_*.md`）

- **存哪**：SQLite 新表 `student_profile(student_id PK, payload TEXT JSON, updated_at)`，沿用 `store.add_diagnosis` 的 JSON payload 範式（`store.py:187`），零 migration。
- **payload**：`interests`（重用 `scaffold.VOCAB` 的 `cat` 計數，`scaffold.py:49`）、`mastered_vocab`/`learning_vocab`（詞頻 × `asr_confidence≥0.8` × AI 回應無糾錯 三重交叉）、`error_patterns`（把 `diagnose._detect_patterns` 的逐批旗標**升級成跨時間累積 + trend**）、`difficulty`（B3 CEFR 的種子）。
- **抽取**：新檔 `server/profile.py` `build_profile(interactions, diagnoses, prev)`，純標準庫、規則式起步、重用 `scaffold._find_zh_vocab`/`diagnose._chinese_ratio`/`_english_word_count`/`_has_article_correction`；可選 `refine_with_llm` 走雙軌精煉。
- **不需向量資料庫**：規模小（單學生數十~數百筆、固定 50 詞庫）、檢索型態是聚合統計非語意相似、邊緣 CPU/RAM 吃緊。跨越門檻（開放語料數千筆 + 語意 RAG）才重評，屆時先上 SQLite FTS5。
- **接點**：`store.py:98 init_db` 加建表 + `get_profile`/`save_profile`；`app.py:139-147` cloud sync 區塊 3 行（build→save，與診斷同一異步時機）。
- **待決風險**：「已掌握」判定偏寬（asr_confidence≠語言掌握，靠 AI 無糾錯交叉 + LLM 精煉收斂）；詞庫外盲區；更新時機若全程 edge 不會刷新（需 seed 預建 + 每 N 輪觸發）。

## 六、B3 CEFR 難度階梯 + 主題迭代（詳見 `b_axis/B3_*.md`）

- **新檔 `server/curriculum.py`**（stdlib-only，可自我驗證）：四維 → **CAS**（加權平均，fluency 0.30/vocab 0.25/grammar 0.25/pron 0.20）→ **Band 1–5**（`BAND_CUTS=[35,50,65,80]` + 最弱維度 gate 防跛腳升級）× **Step 1–5**（band 內線性五等分）＝ `level_index` 1–25。
- **分級錨點**（web 已確認）：劍橋 YLE Starters/Movers/Flyers ↔ CEFR pre-A1/A1/A2；台灣 108 課綱國小主戰場 **pre-A1→A1**，A2 為天花板。
- **升降級（含遲滯）**：`move ∈ promote/hold/lateral/demote`。慢升（跨 band 需連 2 窗、單窗最多 +1 step）、快降護信心（對外包裝成「換個玩法」）、疲態橫移只換主題不動 level。掛回 `diagnose._detect_patterns` 三旗標 + `emotional_status` 趨勢。
- **主題庫**：直接用 `scaffold.VOCAB` 的 6 類 `cat` + `_EXTENSION_QUESTIONS`（`scaffold.py:152`），標註解鎖 band；掌握/疲態/興趣三種驅動迭代（興趣吃 B2 的 `interests`）。
- **介面**：B3 產 `level_state`（掛進 diagnosis，向後相容），把 `directive_hint`/`target_form`/`focus` 交給 **B1** 拼成 `companion_directive`（**與 B1 §4.2 同一接法 A**）。B3 只產資料，不碰 `llm.py` prompt 本體。
- **待決風險**：**沉默訊號拿不到**——pipeline 低信心 fallback 不寫 DB（`pipeline.py:182`），需拍板走 A（`_detect_patterns` 加偵測「不會/I don't know」放棄語，低侵入）或 B（動 pipeline/store 記真沉默）；「太空」等新主題無 `VOCAB` 詞條需另補；CAS 權重/門檻需用 `seed_demo` 阿明曲線校準。
